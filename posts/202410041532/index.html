github를 이용하다보면 개인계정, 회사계정 등등 여러 계정을 사용하는 상황이 오곤 한다.
이 글은 macOS의 터미널에서 각 레포지토리별 다른 github 계정을 사용할 때, SSH를 이용하는 방법을 설명한다.

새 SSH 키 생성 및 ssh-agent에 추가하기

1. SSH 키 생성

1. 키 생성

우선 터미널을 열고 다음을 입력한다.

1

ssh-keygen -t ed25519 -C "your_email@example.com"

 

  • -t : 이 옵션은 생성할 키의 유형을 지정하는데 사용한다. 일반적으로 어떤 암호화 알고리즘을 사용할지에 따른다.
  • -C : 이 옵션은 키에 추가할 주석을 지정하는데 사용된다. 여러 키를 가지고 있을 경우, 키를 식별할 때 유용하며, 보통 이메일 주소나 설명을 넣는다.

 

만약 ed25519 방식을 지원하지 않는 레거시 시스템을 사용하는 경우 다음과 같이 rsa 방식을 사용할 수 있다.

1

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

 

  • -b : 이 옵션은 키의 비트 길이(bit length)를 지정하는데 사용된다. RSA 키의 경우, 일반적으로 2048비트가 기본값이지만, 보안을 강화하기 위해 따로이 -b 4096명령을 추가하여 더 긴 키를 사용할 수 있다. 비트가 길어지면 키 생성 및 사용시 약간 더 많은 연산 자원을 필요로 한다.
ssh-keygen -t 옵션 종류
  • rsa: 오래된 방식이지만 여전히 많이 사용됨. ssh-keygen -t rsa로 생성.
  • ecdsa: ECC(Elliptic Curve Cryptography)를 기반으로 하여, 작은 키 크기로도 높은 보안을 제공함.
  • ed25519: 가장 최신의 알고리즘 중 하나로, 빠르고 높은 보안을 제공함. 현대적인 서버에서 많이 권장됨.

 

2. 파일 저장위치 및 파일명 선택

1. 키 생성같이 실행을 했다면 다음과 같은 문구가 뜬다.

1
2
3

> Generating public/private ALGORITHM key pair.

> Enter a file in which to save the key (/Users/YOU/.ssh/id_ALGORITHM): [Press enter]

 

두번째 문구는 키를 저장할 위치를 묻고 있는데 아무 입력 없이 enter를 누르면 위 문구에 나온대로 .ssh 위치와 id_ALGORITHM이라는 파일명으로 저장된다. 만약 위치와 파일명을 바꾸고 싶다면 이 물음에서 같은 양식으로 작성한 후, enter를 누르면 된다. ex) /Users/YOU/Documents/ssh/id_IamSSH

 

3. SSH 키 보안암호(passphrase) 입력

2. 파일 저장위치 및 파일명 선택 이후에는 다음과 같은 문구가 나온다.

1
2

> Enter passphrase (empty for no passphrase): [Type a passphrase]
> Enter same passphrase again: [Type passphrase again]

생성한 SSH 키에 대한 암호를 설정하는 화면으로 암호 생성을 무시하고 싶다면 아무것도 입력하지 않은 상태에서 그대로 enter를 누르면 된다.

SSH키의 암호 사용 권장
  1. SSH 키의 생성: • SSH 키를 생성하면 **private key(개인 키)**와 public key(공개 키) 한 쌍이 만들어진다. • Public key는 서버에 저장되고, private key는 로컬에 보관된다. • 서버는 접속 시 public key를 사용해 보낸 메시지가 private key와 일치하는지 확인하고, 이 방식으로 인증이 이루어진다다.
  2. 파일 탈취: • Private key는 매우 중요한 파일이다. 탈취되면 해당 키를 사용하는 모든 서버에 대한 접근 권한이 공격자에게 넘어갈 수 있다. • Public key는 공개되어도 문제가 없지만, private key가 탈취되면 누구나 이 키를 사용하여 서버에 접속할 수 있게 된다.

이로써 SSH key를 생성하는 부분은 끝이 났다.

 

2. ssh-agent에 SSH 키 추가 

1
2

$ eval "$(ssh-agent -s)"  
> Agent pid 59566

$ eval "$(ssh-agent -s)" 입력시 아래 나오는 Agent pid는 디바이스마다 또는 실행될때마다 달라질 수 있기에 신경쓰지 않아도 된다.

macOS의 경우 사실 위와 같이 따로이 ssh-agent를 따로이 구동시켜줄 필요가 없다. ssh key를 등록하는 과정에서 ssh-agent가 백그라운드에서 항상 자동으로 실행되고 있기 때문이다. 다만 재부팅시 agent에 등록된 key들은 휘발되기에 다시 등록해줘야 하는 과정이 필요하다.

 

1. ssh/config 파일 편집

먼저 ~/.ssh/config 파일이 기본 위치에 있는지 확인한다.

1

$ open ~/.ssh/config

 

만약 다음과 같이 파일이 없다고 나온다면,

>The file /Users/YOU/.ssh/config does not exist.

아래 명령어로 파일을 생성한다.

1

touch ~/.ssh/config

 

그리고 해당 파일을 사용하는 코드 편집기로 연다. 다음 예시는 vim을 이용해 해당 파일을 열고 있다.

1

vim ~/.ssh/config

 

해당 파일에 다음과 같이 입력한다.

 1

Host github.com-personal
  HostName github.com
  AddKeysToAgent yes
  UseKeychain yes
  User ABCD
  IdentitiesOnly yes
  IdentityFile ~/.ssh/id_ed25519

Host github.com-work
  HostName github.com
  AddKeysToAgent yes
  UseKeychain yes
  User ABCD-work
  IdentitiesOnly yes
  IdentityFile ~/.ssh/id_ed25519_work

 

옵션 설명

  • Host : SSH 접속시 사용할 호스트 설정상 주소이다.
  • HostName : 실제로 접속할 원격 호스트의 주소를 따로이 지정할 때 사용한다. Host에서 계정별로 또는 사용처별로 나눈 가상의 별칭을 추가했다면 (github.com-personal, github.com-work 처럼) 여기에 따로이 정확한 HostName으로 github.com이라고 구분해놓을 수 있다. Host와 실제 원격호스트가 같을 경우 HostName은 빼도 무방하다.

 

Note

만약 다계정이 아닌 단일계정에 대한 SSH 키 설정이라면 다음과 같이 Host 별칭을 주지않고 HostName 옵션을 제거한 후 진행하면 된다.

1
2
3
4
5

Host github.com
  AddKeysToAgent yes
  UseKeychain yes
  User ABCD
  IdentityFile ~/.ssh/id_ed25519

 

  • AddKeysToAgent yes : SSH 에이전트에 SSH 키를 추가할지 여부를 지정한다. SSH 에이전트는 재부팅시 추가된 모든 키들이 휘발되는 특성이 있다. 때문에 이 설정을 해두면 재부팅시 에이전트가 실행됨과 동시에 키를 자동으로 추가한다.
  • UseKeychain yes : SSH 키에 대한 암호(passphrase)가 있는 경우 이 암호를 매번 입력하지 않아도 되도록 macOS의 키체인에 넣어놓는다. 에이전트가 실행되고 해당 키가 에이전트에 등록되어있을 경우, SSH 클라이언트는 해당 키에 대한 암호를 사용자에게 묻지않고 키체인에서 가져다 쓴다. SSH키에 대한 암호가 없다면 이 옵션은 생략한다.
  • User ABCD : SSH를 통해 원격서버에 접속할 때 사용할 사용자 이름을 지정한다. github의 경우 계정명을 입력하면 된다. (ex. github 계정이 만약 inchan이라면 User inchan 이라고 입력하면 된다.) 다계정일 경우 이 User 명에 따라 정해진 SSH 키를 사용함으로 정확히 구분하여 입력한다.
  • identitiesOnly yes : SSH가 IdentityFile로 지정된 키 파일만을 사용하도록 강제하는 옵션이다. 기본적으로 SSH 는 여러 키를 자동으로 시도할 수 있는데, 이 옵션을 사용하면 명시적으로 지정된 키만을 사용한다. 다계정에서 각각 다른 SSH키 사용시 이 옵션 설정을 추천한다.
  • IdentityFile ~/.ssh/id_ed25519 : 사용할 개인 키 파일을 지정한다. 기본적으로 경로와 파일명이다.

 

Question
지금 예시에서는 왜 id_ABCD와 같이 User 명이 아니라 어렵고 복잡하게 이런 이상한 이름을 썼을까??
추후 설명할 AddKeysToAgent yes 옵션 때문인데 이 글에서 더 자세히 알아볼 수 있다.

 

2. ssh-agent에 key 추가하기

키를 만들었으니 이제 내가 이 키를 사용하겠다는 뜻으로 등록을 해야한다.

1

ssh-add --apple-use-keychain ~/.ssh/id_ed25519

위와 같은 경로 및 파일명으로 키를 등록한다.

  • –apple-use-keychain은 해당 키에 대한 암호(passphrase)를 앞서 언급한 키체인에 등록하여 사용하겠다는 뜻이다.1

 

정상적으로 등록되었는지 확인하는 방법은 다음과 같다.

1

ssh-add -l

 

3. github에 SSH key 등록하기

github에는 아까 만든 한 쌍의 SSH 키 중 public 키를 등록한다.
다음 명령어는 터미널에서 public 키의 내용을 복사한다.

1

$ pbcopy < ~/.ssh/id_ed25519.pub

 

만약 키 복사가 제대로 되지 않는 경우 경로를 재확인해본다.
위 명령어는 기본설정대로 키 파일을 저장했을 경우인 ~/.ssh 폴더내로 경로가 잡혀있다.

등록절차는 매우 간단하다. github website에서 해당 계정으로 로그인 한 후,
settings - SSH and GPG keys - SSH keys 에서 우상단에 보이는 New SSH key를 누른다.

 

New SSH key 버튼을 누르면 다음 화면을 볼 수 있다.

 

  • Title : 나중에 github에 여러 SSH키가 입력되어있을 경우 지금 등록하는 SSH 키를 구분할 수 있도록 적당한 제목을 입력한다. (ex. home-my-macbook)
  • Key type : Authentication Key
  • Key : 방금 전 터미널에서 복사해둔 퍼블릭 키의 내용을 붙여넣는다.

그리고 마지막으로 Add SSH key를 눌러 등록하면 끝!

 

4. SSH - Host 테스트하기

이제 잘 작동하는지 테스트해본다.

1

ssh -T git@github.com-personal

ssh/config 파일에서 설정한 옵션 Host 부분의 별칭을 주소로 한다. 만약 Host 별칭을 github.com-work로 했다면 ssh -T git@github.com-work라고 입력하면 Host에 따라 테스트할 수 있다.
정상적인 경우, “Hi 사용자명 ~~ 어쩌구 저쩌구 “와 같은 문구를 볼 수 있다.

 

Warning
SSH key를 이용해 깃 레포지토리를 관리하게 되었다면, 이제부턴 해당 계정을 이용하는 git clone 및 기존 레포에 원격 주소 할당시 github/user/repository의 주소복사시 https 주소가 아닌 ssh 주소를 이용해야 한다!!
