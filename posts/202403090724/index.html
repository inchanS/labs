문제상황

브라우저에서 chatGPT 프롬프트의 입력창에 글을 쓰고 아무리 “엔터"를 눌러도 반응이 가질 않는 상황이 생겼다.
일단 의심되는 변화는 macOS를 sonoma 14.4로 업그레이드 한 정황뿐인데 이래저래 만져보다 문제를 찾았다.

문제는 바로 chatGPT의 local language 지원(Alpha) 기능이었다.
(chatGPT 웹사이트의 왼쪽 아래, 자신의 아이디를 클릭하고 나오는 설정에서 확인 가능)
 


 

일단 이 기능은 ‘Auto-detect‘가 default로 설정되어있는데, 만약 이 값을 ‘한국어’ 등으로 바꾸었다면 먹통 문제가 발현되었다.

게다가 한번 바꾸면 이 설정에서 아무리 다른 값을 선택해도 먹히질 않는다.

테스트 해본 결과, Safari, chrome 두 브라우저 모두 동일하게 발생했다.
아마도 Beta도 아닌 Alpha 테스트중인 기능이라 생긴 버그가 아닌가 생각된다.
 


 

현재 chatGPT에서 알파테스트 중인 브라우저 언어지원 목록에는 한국어가 포함되어있지 않다.1

 


 

그런데 아이러니하게도 chatGPT 설정에서 보면 ‘한국어’가 포함되어있어 마치 지원되는 것처럼 선택창에 포함되어있고, 아마도 이러한 부분에서 버그가 발생해 모든 프롬프트 입력이 먹통이 되는 상황이 발생하지 않았나 싶다.
 

이 문제는 2024년 3월 이후 chatGPT의 업데이트에 따라 자연히 사라질 수 있음!
즉, 브라우저 언어지원에 한국어가 포함되거나
또는 Beta 테스트 이후의 단계에 들어서게 되면 언제든 현재 이 문제는 자연히 해결 될 수 있는 한시적 문제일 것으로 보인다.

해결방법

해결 방법은 브라우저의 site 쿠키 등을 저장한 정보를 지워주면 된다.
웹사이트의 쿠키 및 정보를 지우고 나면 다시 로그인해줘야 하는 번거로움이 있지만 한번만 수행하면 되기에 큰 문제는 없었다.

safari에서 site 정보 지우는 방법

  1. safari 브라우저의 설정에서 “개인정보 보호 / 웹사이트 데이터 관리… " 메뉴를 클릭한다.
  2. “openai"를 검색해 해당 사이트의 데이터를 제거한다.
  3. 다시 chatGPT 사이트에 들어가 로그인하면 끝.
chatGPT 상단에 나타나는 배너 조심

이게 간헐적으로 나타나는지 모르겠지만 새로 로그인 하고 chatGPT 페이지에 접속하면 상단에

chatGPT local 언어 지원에 참여하시겠습니까? ‘예’

뭐 이런 문구의 배너가 뜰때가 있는데 이 배너에서 ‘예’를 선택하면 자동으로 위에서 언급한 local language 설정이 한국어(또는 해당 브라우저의 언어설정에 따라)로 적용되면서 문제가 발생한다.
